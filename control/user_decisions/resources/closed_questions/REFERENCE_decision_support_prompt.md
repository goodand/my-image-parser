# Decision Support Prompt Reference

## Purpose

재사용 가능한 의사결정 지원 프롬프트를 보관한다.

이 프롬프트는 다음 상황을 위한 기본 템플릿이다.

- 결정을 폐쇄형 질문으로 좁히고 싶을 때
- 상위 철학이나 상위 전략 수준에서 trade-off를 강제로 드러내고 싶을 때
- 내가 핵심 개념을 충분히 이해했는지 점검하고 싶을 때
- 아직 판단이 어려우면 무엇을 더 알아야 하는지 추천받고 싶을 때

## Prompt

```md
나의 의사결정을 지원해줘

(1) 내가 결정하기 좋도록 폐쇄형 질문 형식인지,
(2) 어려운 개념의 사항들 중 상위 철학이나 상위 전략으로 결정할 수 있다면 그러한 가치관 중에서 어떠한 것을 결정해야 하는지 (기존에 이미 이해하고 있는 영역에서 무엇을 포기할 것인지 선택 강요 전략)
(3) 내가 핵심 개념을 충분히 이해를 했는지 확인을 위한 질문
(4) 그럼에도 결정이 어렵다면 결정을 위해서 추가적으로 알아야 하는 핵심 개념이 무엇인지 추천
```

## Extended Prompt Variant

질문 간 의존관계가 있거나, 어려운 개념 설명이 먼저 필요하거나, 내가 답한 내용을 다시 상위 결정으로 매핑해야 하는 경우에는 아래 확장형을 쓴다.

```md
나의 의사결정을 지원해줘

(1) 내가 결정하기 좋도록 폐쇄형 질문 형식인지,
(2) 어려운 개념의 사항들 중 상위 철학이나 상위 전략으로 결정할 수 있다면 그러한 가치관 중에서 어떠한 것을 결정해야 하는지 (기존에 이미 이해하고 있는 영역에서 무엇을 포기할 것인지 선택 강요 전략)
(3) 내가 핵심 개념을 충분히 이해를 했는지 확인을 위한 질문
(4) 그럼에도 결정이 어렵다면 결정을 위해서 추가적으로 알아야 하는 핵심 개념이 무엇인지 추천
(5) 질문 간 의존관계나 개념 설명이 복잡하면 표, 트리, Mermaid 같은 가장 단순한 시각화로 설명
(6) 내가 이미 답한 내용을 상위 결정표나 규칙 매핑으로 다시 정리해서 남은 질문 수를 줄여줘
```

## Expected Response Shape

1. 폐쇄형 질문
2. 상위 철학 또는 상위 전략 기준의 선택 강요 질문
3. 핵심 개념 이해 확인 질문
4. 추가로 알아야 할 핵심 개념 추천

필요 시 추가:

5. 간단한 시각화 또는 결정 구조도
6. 이미 답한 내용의 상위 결정 매핑과 남은 질문 축소

## Observed Conversation Pattern

참조 대화에서는 다음 패턴이 반복되었다.

1. 질문 품질 진단
2. 상위 철학 선택으로 하위 질문 축소
3. 이해가 부족한 개념 설명
4. 필요 시 비유, 코드 예시, 간단한 구조 시각화 제공
5. 사용자의 답변을 다시 결정표로 매핑
6. 남은 질문만 다시 폐쇄형으로 제시

즉, 질문지는 독립 도구라기보다 `폐쇄형 질문 + 개념 브리지 + 선택 강요 + 시각화 보조 + 재매핑`이 결합된 대화 프로토콜에 가깝다.

## Visualization Trigger

다음 경우에는 시각화를 함께 제안하는 것이 좋다.

- 질문들이 서로 의존해서 독립적으로 답하면 모순이 생길 때
- 사용자가 `이해하기 어려움`, `더 설명이 필요해`처럼 개념 부족을 직접 드러낼 때
- 상위 철학 선택이 하위 질문 여러 개를 한 번에 결정할 수 있을 때
- 이미 답한 내용을 다시 구조적으로 매핑해야 할 때

## Usage Notes

- 열린 브레인스토밍보다 결정 가능성을 높이는 질문을 우선한다.
- trade-off가 있으면 "무엇을 얻을지"보다 "무엇을 포기할지"를 먼저 드러낸다.
- 사용자가 이미 이해한 개념과 아직 모르는 개념을 분리해서 질문한다.
- 답변은 decision memo, ADR, closed-question inbox로 쉽게 옮길 수 있게 짧고 구조적으로 만든다.
- 시각화는 별도 도구가 아니라 질문 흐름을 보조하는 설명 surface로 취급한다.
